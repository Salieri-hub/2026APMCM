# Codex 项目协作规范

你是本项目的 AI 开发助手，请遵循以下工作流程。

## 一、开始工作

每次开始工作时，请先阅读以下文件（如果存在）：

1. README.md
2. AGENTS.md
3. AI_CONTEXT.md
4. TODO.md
5. doc/ 目录下相关文档

阅读完成后，概括当前项目状态，包括：

* 当前项目目标
* 已完成工作
* 当前正在进行的任务
* 下一步建议
* 需要特别注意的约束

不要立即修改代码，先确认已经理解项目。

---

## 二、开发规范

开发过程中：

* 尽量保持现有代码风格一致。
* 不随意修改已经稳定运行的模块。
* 修改前先理解相关代码逻辑。
* 尽量小范围修改，不进行无关重构。
* 新增函数和模块添加必要注释。
* 保持目录结构整洁。

---

## 三、完成任务后

完成一个阶段性任务后，请更新以下文档：

### AI_CONTEXT.md

更新内容包括：

* 当前项目整体进度
* 本次完成内容
* 修改了哪些模块
* 当前模型或算法状态
* 遇到的问题
* 下一步计划
* 需要提醒其他开发者注意的事项

要求内容保持最新。

---

### TODO.md

同步更新：

* 已完成任务
* 未完成任务
* 新增待办事项
* 当前优先级

---

### README.md（必要时）

如果新增：

* 模块
* 环境依赖
* 启动方式
* 数据格式
* 使用流程

请同步更新 README.md。

---

## 四、Git 协作流程

默认工作流程如下：

开始开发：

git pull

完成开发：

git add .
git commit -m "简要说明本次修改"
git push

不要建议跳过上述流程。

---

## 五、多人协作原则

如果发现：

* 代码与文档不一致
* AI_CONTEXT.md 未更新
* TODO.md 已过期

请优先提醒并协助维护这些文档，而不是继续开发。

---

## 六、回答原则

回答时优先依据：

AI_CONTEXT.md > AGENTS.md > README.md > 当前代码

如果发现文档与代码冲突，请指出冲突位置，并建议更新文档。

在不了解完整项目背景时，不要凭空假设项目结构，应先阅读项目文件再继续工作。
